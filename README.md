# راهنمای فنی بات تاس‌انداز بله (Bale)

## ۱. مقدمه

این سند به تشریح ساختار و عملکرد یک بات ساده برای پیام‌رسان «بله» می‌پردازد. وظیفه اصلی این بات، دریافت پیام از کاربر و ارسال یک عدد تصادفی بین ۱ تا ۶ (شبیه‌سازی انداختن تاس) در پاسخ است. کد بات با استفاده از زبان **پایتون** و با رویکرد **Polling (نظرسنجی)** پیاده‌سازی شده است.

هدف اصلی این راهنما، ارائه یک دید کلی از اجزای اصلی بات، جریان کاری آن و مفاهیم کلیدی مرتبط با API بله است.

---

## ۲. اجزای اصلی کد

کد بات از چند بخش کلیدی تشکیل شده است:

### کتابخانه‌های مورد نیاز
- **`requests`**: برای ارسال درخواست‌های HTTP به API بله (دریافت پیام‌ها و ارسال پاسخ).
- **`random`**: برای تولید عدد تصادفی جهت شبیه‌سازی تاس.

### متغیرهای اصلی
- **`TOKEN`**: یک رشته منحصربه‌فرد که به عنوان شناسه و کلید احراز هویت بات شما عمل می‌کند. **این توکن باید کاملاً محرمانه باقی بماند.**
- **`URL`**: آدرس پایه API بله که با توکن بات ترکیب می‌شود. تمام درخواست‌ها به این آدرس ارسال می‌گردند.
- **`offset`**: یک متغیر عددی که برای مدیریت پیام‌های خوانده‌شده استفاده می‌شود. این متغیر تضمین می‌کند که بات یک پیام را دو بار پردازش نکند.

### حلقه اصلی (`while True`)
قلب تپنده بات یک حلقه بی‌نهایت است که به طور مداوم برای دریافت پیام‌های جدید، سرور بله را چک می‌کند. این رویکرد **Polling** نام دارد.

---

## ۳. جریان کاری بات (Workflow)

روند کار بات به صورت مرحله به مرحله به شرح زیر است:

1.  **شروع و مقداردهی اولیه**:
    - اسکریپت اجرا می‌شود.
    - متغیر `offset` با مقدار اولیه `0` تنظیم می‌شود.

2.  **ورود به حلقه بی‌نهایت**:
    - بات وارد حلقه `while True` می‌شود و مراحل زیر را به صورت مداوم تکرار می‌کند.

3.  **دریافت آپدیت‌ها (Get Updates)**:
    - بات یک درخواست `GET` به متد `getUpdates` در API بله ارسال می‌کند.
    - پارامتر `offset` در این درخواست استفاده می‌شود تا فقط پیام‌هایی که بعد از آخرین پیام پردازش‌شده ارسال شده‌اند، دریافت شوند.
    *   **مثال درخواست**: `https://tapi.bale.ai/bot{TOKEN}/getUpdates?offset=12345`

4.  **پردازش پیام‌ها**:
    - پاسخ دریافت شده از سرور (که با فرمت JSON است) بررسی می‌شود.
    - بات در میان لیست پیام‌های جدید (`result`) حلقه می‌زند.
    - برای هر پیام:
        - **به‌روزرسانی `offset`**: مقدار `update_id` پیام فعلی به اضافه یک، در متغیر `offset` ذخیره می‌شود. این کار باعث می‌شود در درخواست بعدی، این پیام دوباره دریافت نشود.
        - **استخراج `chat_id`**: شناسه یکتای گفتگو (خصوصی، گروه یا کانال) از پیام استخراج می‌شود تا بات بداند پاسخ را به کجا ارسال کند.

5.  **اجرای منطق اصلی (انداختن تاس)**:
    - با استفاده از تابع `random.randint(1, 6)`، یک عدد صحیح تصادفی بین ۱ تا ۶ تولید می‌شود.

6.  **ارسال پاسخ (Send Message)**:
    - بات یک درخواست `POST` به متد `sendMessage` در API بله ارسال می‌کند.
    - بدنه این درخواست یک آبجکت JSON شامل موارد زیر است:
        - `chat_id`: شناسه‌ای که در مرحله ۴ استخراج شد.
        - `text`: متن پیام پاسخ، که شامل نتیجه تاس است. (مثلاً: `🎲 عدد تاس: 4`)

7.  **تکرار**:
    - حلقه به ابتدای خود بازگشته و دوباره منتظر پیام‌های جدید می‌ماند.
